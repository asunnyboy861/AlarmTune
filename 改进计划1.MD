# AlarmTune 改进计划 v1.0

> 基于指南.md 和 us.md 的功能要求，对当前代码进行全面评估后制定的优化方案。
> 遵循：单一职责、高内聚低耦合、代码复用、最小代码原则、风格一致性、继承优先原则。

---

## 一、当前问题诊断

### 🔴 P0 - 严重缺陷（必须修复）

| 编号 | 问题 | 影响 | 根因 |
|------|------|------|------|
| P0-1 | **重复日期选择器缺失** | 用户无法设置重复闹钟（如工作日、周末），核心功能不完整 | AlarmEditView 中缺少 repeatDays 的 UI 组件 |
| P0-2 | **repeatDays 数据未持久化** | 即使 UI 添加了选择器，保存时也未写入 CoreData | AlarmViewModel.addAlarm/updateAlarm 未处理 repeatDays 参数 |
| P0-3 | **AlarmScheduler 未处理重复日期** | 当前 scheduleAlarm 使用 repeats=true 但无 weekday 区分，所有闹钟每天都响 | 未根据 repeatDays 创建多个 UNCalendarNotificationTrigger |

### 🟡 P1 - 体验优化（建议修复）

| 编号 | 问题 | 影响 | 根因 |
|------|------|------|------|
| P1-1 | **音量滑块拖动时无实时预览** | 用户调整音量时无法即时听到效果，体验不佳 | VolumeSliderView.onChange 仅触发 haptic，未调用 onPreview |
| P1-2 | **缺少"仅振动"模式** | 指南要求支持 0 音量 + 仅振动场景 | 无专门开关，用户需手动将音量拖到 0 |
| P1-3 | **SettingsView 测试音频说明不够清晰** | 审核人员可能找不到测试入口 | 缺少引导性文案和步骤说明 |
| P1-4 | **AudioService.urlForSound 过于复杂** | 代码冗余，维护困难 | 多次尝试不同路径和扩展名，逻辑重复 |

### 🟢 P2 - 代码整洁（可选优化）

| 编号 | 问题 | 影响 | 根因 |
|------|------|------|------|
| P2-1 | **AlarmEditView State 初始化重复** | init 中大量 State 初始化代码，可读性差 | 未使用统一配置方法 |
| P2-2 | **部分常量未纳入 AppConstants** | 分散的魔法数字 | 如 fadeInDuration range 1...30 等 |

## 二、优化方案详述

### 模块 1：重复日期选择器（DayPickerView）

#### 1.1 需求描述
- 在 AlarmEditView 的"Time" Section 下方添加重复日期选择
- 支持选择周一到周日（多选）
- 未选择任何日期时表示"仅一次"
- 已选日期在 AlarmRowView 中显示为 "M T W T F S S" 格式
- 与现有 AlarmItem.repeatDays 字段（[Int]?）兼容

#### 1.2 生成规则

**文件位置**：`Views/Components/DayPickerView.swift`

**设计原则**：
- 单一职责：只负责日期选择 UI，不涉及业务逻辑
- 复用现有风格：使用 AlarmRowView/VolumeSliderView 相同的圆角、阴影、配色风格
- 继承优先：使用 SwiftUI 原生组件，不引入第三方库

**接口设计**：
```swift
struct DayPickerView: View {
    @Binding var selectedDays: [Int]  // 0=Sun, 1=Mon, ... 6=Sat
    var onChange: (() -> Void)?
    
    // 内部常量
    private let daySymbols = ["S", "M", "T", "W", "T", "F", "S"]
    private let fullDayNames = ["Sun", "Mon", "Tue", "Wed", "Thu", "Fri", "Sat"]
}
```

**UI 规范**：
- 水平排列 7 个圆形按钮
- 选中状态：accentColor 背景 + 白色文字
- 未选中状态：gray.opacity(0.1) 背景 + secondary 文字
- 尺寸：iPhone 32pt 圆形，iPad 44pt 圆形
- 间距：8pt
- 下方显示当前选择文案：如 "Every Day" / "Weekdays" / "Weekends" / "Once"

**代码编写示例**：
```swift
import SwiftUI

struct DayPickerView: View {
    @Binding var selectedDays: [Int]
    @Environment(\.horizontalSizeClass) private var horizontalSizeClass
    var onChange: (() -> Void)?

    private let daySymbols = ["S", "M", "T", "W", "T", "F", "S"]
    private let fullDayNames = ["Sun", "Mon", "Tue", "Wed", "Thu", "Fri", "Sat"]

    var body: some View {
        VStack(spacing: isPad ? 12 : 8) {
            HStack(spacing: isPad ? 12 : 8) {
                ForEach(0..<7, id: \.self) { index in
                    dayButton(for: index)
                }
            }

            Text(selectionDescription)
                .font(.system(size: descriptionFontSize))
                .foregroundColor(.secondary)
        }
    }

    private func dayButton(for index: Int) -> some View {
        let isSelected = selectedDays.contains(index)
        return Button {
            toggleDay(index)
            onChange?()
            HapticService.shared.selection()
        } label: {
            Text(daySymbols[index])
                .font(.system(size: dayFontSize, weight: .semibold))
                .frame(width: dayButtonSize, height: dayButtonSize)
                .background(isSelected ? Color.accentColor : Color.gray.opacity(0.15))
                .foregroundColor(isSelected ? .white : .secondary)
                .clipShape(Circle())
        }
        .buttonStyle(.plain)
        .accessibilityLabel(fullDayNames[index])
        .accessibilityHint(isSelected ? "Selected" : "Not selected")
    }

    private func toggleDay(_ index: Int) {
        if selectedDays.contains(index) {
            selectedDays.removeAll { $0 == index }
        } else {
            selectedDays.append(index)
            selectedDays.sort()
        }
    }

    private var selectionDescription: String {
        switch selectedDays.sorted() {
        case []: return "Once"
        case [0, 6]: return "Weekends"
        case [1, 2, 3, 4, 5]: return "Weekdays"
        case [0, 1, 2, 3, 4, 5, 6]: return "Every Day"
        default:
            let names = selectedDays.sorted().map { fullDayNames[$0] }
            return names.joined(separator: ", ")
        }
    }

    // MARK: - Adaptive Layout
    private var isPad: Bool { horizontalSizeClass == .regular }
    private var dayButtonSize: CGFloat { isPad ? 44 : 32 }
    private var dayFontSize: CGFloat { isPad ? 16 : 12 }
    private var descriptionFontSize: CGFloat { isPad ? 14 : 12 }
}
```

### 模块 2：重复日期调度（AlarmScheduler 增强）

#### 2.1 需求描述
- 有 repeatDays 时：为每个选中的日期创建一个 UNCalendarNotificationTrigger（weekday 匹配）
- 无 repeatDays 时：创建一次性通知（repeats: false）
- 取消时：取消该闹钟的所有相关通知（包括各日期和贪睡）

#### 2.2 生成规则

**修改文件**：`Services/AlarmScheduler.swift`

**scheduleAlarm 逻辑重构**：
```swift
func scheduleAlarm(_ alarm: AlarmItem) {
    guard alarm.isEnabled else { return }

    let content = createNotificationContent(for: alarm)
    
    if let repeatDays = alarm.repeatDays, !repeatDays.isEmpty {
        // 为每个选中的星期几创建独立通知
        for day in repeatDays {
            var dateComponents = DateComponents()
            dateComponents.hour = Int(alarm.hour)
            dateComponents.minute = Int(alarm.minute)
            dateComponents.weekday = day + 1  // UN 使用 1=Sunday
            
            let trigger = UNCalendarNotificationTrigger(dateMatching: dateComponents, repeats: true)
            let requestId = "\(alarm.wrappedId)-day\(day)"
            let request = UNNotificationRequest(identifier: requestId, content: content, trigger: trigger)
            notificationCenter.add(request)
        }
    } else {
        // 一次性闹钟
        var dateComponents = DateComponents()
        dateComponents.hour = Int(alarm.hour)
        dateComponents.minute = Int(alarm.minute)
        
        let trigger = UNCalendarNotificationTrigger(dateMatching: dateComponents, repeats: false)
        let request = UNNotificationRequest(identifier: alarm.wrappedId, content: content, trigger: trigger)
        notificationCenter.add(request)
    }
}
```

**cancelAlarm 逻辑重构**：
```swift
func cancelAlarm(_ alarm: AlarmItem) {
    var identifiers = [alarm.wrappedId]
    
    // 取消所有重复日期的通知
    if let repeatDays = alarm.repeatDays {
        for day in repeatDays {
            identifiers.append("\(alarm.wrappedId)-day\(day)")
        }
    }
    
    // 取消贪睡通知
    identifiers.append("\(alarm.wrappedId)-snooze")
    
    notificationCenter.removePendingNotificationRequests(withIdentifiers: identifiers)
    notificationCenter.removeDeliveredNotifications(withIdentifiers: identifiers)
}
```

**测试用例**：
```swift
// 1. 设置周一到周五 8:00 的闹钟，应创建 5 个 UNNotificationRequest
// 2. 每个 request 的 identifier 格式为 "{alarmId}-day{1..5}"
// 3. 取消该闹钟时，所有 5 个 request 都被移除
// 4. 一次性闹钟的 identifier 就是 alarmId
// 5. 切换闹钟开关时，schedule/cancel 行为正确
```

### 模块 3：音量实时预览（VolumeSliderView 增强）

#### 3.1 需求描述

- 避免过于频繁的播放，使用节流（throttle）机制

#### 3.2 生成规则

**修改文件**：`Views/Components/VolumeSliderView.swift`

**实现方式**：
- 使用 `@State private var lastPreviewTime: Date = .distantPast`
- 在 `.onChange(of: volume)` 中添加节流逻辑
- 节流间隔：0.3 秒

```swift
@State private var lastPreviewTime: Date = .distantPast

// 在 Slider 的 onChange 中：
.onChange(of: volume) {
    HapticService.shared.selection()
    
    let now = Date()
    if now.timeIntervalSince(lastPreviewTime) > 0.3 {
        lastPreviewTime = now
        onPreview?(volume)
    }
}
```

### 模块 5：AudioService 音频路径优化

#### 5.1 需求描述
- 简化 urlForSound 方法，去除冗余逻辑
- 统一音频文件查找策略
- 确保 .aiff 文件能被正确加载

#### 5.2 生成规则

**修改文件**：`Services/AudioService.swift`

**重构后的 urlForSound**：
```swift
private func urlForSound(_ name: String) -> URL? {
    let sanitizedName = name.replacingOccurrences(of: " ", with: "")
    let candidates = [sanitizedName, name]
    let extensions = ["caf", "mp3", "aiff", "wav", "m4a"]
    let directories: [String?] = ["Sounds", nil]
    
    for dir in directories {
        for candidate in candidates {
            for ext in extensions {
                if let url = Bundle.main.url(
                    forResource: candidate,
                    withExtension: ext,
                    subdirectory: dir
                ) {
                    return url
                }
            }
        }
    }
    
    // 检查导入的自定义音频
    let importedDir = FileManager.default
        .urls(for: .documentDirectory, in: .userDomainMask)
        .first?
        .appendingPathComponent("ImportedSounds", isDirectory: true)
    
    if let dir = importedDir {
        for ext in extensions {
            let url = dir.appendingPathComponent("\(sanitizedName).\(ext)")
            if FileManager.default.fileExists(atPath: url.path) {
                return url
            }
        }
    }
    
    return nil
}
```

**测试用例**：
```swift

// 2. "Digital Beep" 能找到 Sounds/DigitalBeep.caf
// 3. 不存在的音频返回 nil
// 4. 自定义导入音频能被找到
```
